# What To Open Source

## Open source

- community scanners
- CLI runners
- GitHub Action wrapper
- basic JSON report
- public docs, fixtures, demo
- storefront assets and proof material

## Keep paid

- apply engines
- wider edge-case transform coverage
- deeper presets for repo layouts
- richer report detail and rollout templates
- enterprise-safe checklists
- downloadable preset bundles
- product ZIP delivery artifacts

## Why this split works

If the free version is too weak, it will not spread.

If the public repo already exposes the paid code, the checkout loses most of
its value.

The clean split is:

- free = trust + fit assessment + machine-generated evidence
- paid = apply engine + edge-case coverage + rollout confidence + preset
  structure
