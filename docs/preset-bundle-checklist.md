# Preset Bundle Checklist

Use this before applying the preset bundle.

## Gather first

- your free scan report
- your repo type or layout guess
- your current SQLAlchemy version
- your target SQLAlchemy version
- your normal validation commands

## Good fit

- you already ran the free scan
- you want more repeatable rollout structure
- you want richer report templates and repo presets
- you still plan to validate the repo yourself after changes

## Not what this bundle is for

- custom migration ownership
- custom coding by a person
- full automatic SQLAlchemy 2.0 migration
- production debugging

## Safe use

- keep version control on
- keep backups
- read the included rollout notes before applying changes
- treat failed validation as a real failure, not a partial success
