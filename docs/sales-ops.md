# Sales Ops

Last updated: 2026-04-06

This is the lightweight operator system for running the business without paid
CRM software.

## Source of truth

Use these files together:

- `docs/lead-tracker.md`
- `docs/launch-log.md`
- `docs/kpi-dashboard.md`
- `docs/preset-bundle-checklist.md`

## Weekly operator loop

1. review site traffic and checkout activity
2. update the KPI dashboard
3. update the lead tracker
4. record launch activity and public feedback
5. decide the single highest-leverage doc, proof, or coverage improvement

## Lead stages

- `new`
- `qualified`
- `waiting_on_free_scan`
- `fit_for_paid_pack`
- `fit_for_preset_bundle`
- `closed_won`
- `closed_lost`

## Inbox rules

- reply from the business email only
- ask every inbound lead to run the free scan first
- do not scope custom work in DMs
- move repeated questions into docs instead of rewriting the same answer forever

## What to measure

- visitor to repo clickthrough
- repo to free-scan usage signal
- paid CTA clicks
- orders
- revenue
- refunds
- objections by theme
- unsupported patterns mentioned by buyers
