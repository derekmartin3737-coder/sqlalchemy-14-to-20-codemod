# Support Scope

Last updated: 2026-04-06

`sa20-pack` support is intentionally bounded.

## Free support

Free support is limited to:

- clarifying published docs
- clarifying current supported scope
- clarifying current unsupported patterns

Free support does not include custom repo debugging.

## Paid product support

Paid products do not include consulting. Paid product support is limited to:

- clarifying published scope
- clarifying install and delivery steps
- clarifying how the downloadable product is meant to be used

## Out of scope

- full application debugging
- infrastructure support
- database/data migration execution
- deployment ownership
- on-call style support
- private environment access
- regulated-data handling without a separate formal process

## Security boundary

Do not send secrets, production credentials, or regulated data in ordinary
support requests.

## Non-guarantee

Support helps clarify product scope and usage. It does not guarantee that a
buyer's repo will pass every validation step or behave correctly in production
after changes.

## Best way to request support

Send:

- order reference
- SQLAlchemy version details
- report excerpt
- the unsupported pattern or validation failure
- a minimal sanitized repro if available
