# Package Publisher Hardening Pack Commercial Case

## Boss-Approval Sentence

This pack tells us which package release workflows still rely on long-lived
tokens and which repo-side changes are needed before npm/PyPI trusted publishing
and provenance can work safely.

## Why This Can Be Paid

- Package publishing credentials are high-value supply-chain targets.
- npm and PyPI now have official trusted publishing and attestation paths.
- Many repos still contain old token-based release workflows.
- The buyer gets exact workflow findings and small safe patches.

## Paid SKU Candidate

`Package Publisher Hardening Team Pack`

- local CLI
- JSON and HTML report
- workflow OIDC patch mode
- npm/PyPI workflow templates
- multi-repo summary
- registry setup checklist

## Do Not Sell As

- registry-side configuration automation
- guarantee against supply-chain compromise
- secret scanner
- complete GitHub Actions security scanner
- replacement for npm/PyPI documentation
