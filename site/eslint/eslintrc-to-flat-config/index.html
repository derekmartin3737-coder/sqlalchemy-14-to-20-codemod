How to migrate .eslintrc to eslint.config.cjs with FlatCompat

Convert static .eslintrc JSON or YAML into eslint.config.cjs without executing JS logic.

Search intent

eslintrc to flat config

The honest path is to bridge static config with FlatCompat and stop on JS-backed config.

Direct answer

What changes

If the source config is static JSON or YAML data, the bridge file can be generated deterministically.

Example

Before and after

Before

{
  "extends": ["eslint:recommended"],
  "rules": {"no-console": "warn"}
}

After

const { FlatCompat } = require("@eslint/eslintrc");
const compat = new FlatCompat();

module.exports = [
  ...compat.config({
    extends: ["eslint:recommended"],
    rules: { "no-console": "warn" },
  }),
];

Typical symptoms

  • The repo still has a static .eslintrc file.
  • The team wants a bridge, not a plugin-by-plugin rewrite party.
  • People need a clear stop signal on JS configs.

What the product covers

  • Static JSON and YAML config discovery.
  • FlatCompat bridge generation in the supported subset.
  • Manual-review findings for JS configs and existing flat config files.

Manual-review boundary

  • .eslintrc.js, .cjs, or .mjs logic.
  • Repos that already have eslint.config.*.
  • Ambiguous ignore rules that do not map cleanly.

Keep the migration narrow

Use the exact-problem guides as a triage layer, then decide whether the repo is inside the supported subset.
