ESLint Migration Guides

What to do when eslint.config.* already exists

If the repo already has eslint.config.js, .cjs, or .mjs, the static legacy-config migration path should stop for manual review.

Search intent

existing eslint.config flat config migration

If the repo already has a flat config file, you are not starting from the simple bridge case anymore.

Direct answer

What changes

Treat existing eslint.config.* files as manual review. The next step depends on whether the flat config is authoritative, partial, or mid-migration.

Typical symptoms

  • The repo already contains eslint.config.js, .cjs, or .mjs.
  • The flat-config pack needs to stop before generating duplicate or conflicting output.
  • Teams want fast repo qualification instead of accidental config churn.

What the product covers

  • Explicit existing-flat-config findings.
  • Cross-links to the supported static legacy-config pages.
  • Repo-fit guidance before buying the flat-config pack.

Manual-review boundary

  • Deciding whether the existing flat config is authoritative.
  • Cleaning up mixed legacy and flat-config states.
  • Repos that also have JS-backed legacy configs.

Use the exact-problem guides as a triage layer, then decide whether the repo is inside the supported subset.
