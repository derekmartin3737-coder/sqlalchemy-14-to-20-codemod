What to do when eslint.config.* already exists

If the repo already has eslint.config.js, .cjs, or .mjs, the static legacy-config migration path should stop for manual review.

Start here

The safe first move

Treat existing eslint.config.* files as manual review. The next step depends on whether the flat config is authoritative, partial, or mid-migration.

Stop before automation when: Deciding whether the existing flat config is authoritative. Cleaning up mixed legacy and flat-config states.
