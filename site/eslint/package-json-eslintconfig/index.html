How to migrate package.json eslintConfig to flat config

Convert static package.json eslintConfig blocks into flat config without evaluating JS.

Search intent

package.json eslintConfig flat config

package.json eslintConfig is a good automation target because the source is structured data.

Direct answer

What changes

When the repo exposes one static eslintConfig source in package.json, the bridge file can be generated cleanly.

Example

Before and after

Before

"eslintConfig": {
  "extends": ["eslint:recommended"],
  "env": {"node": true}
}

After

module.exports = [
  ...compat.config({
    extends: ["eslint:recommended"],
    env: { node: true },
  }),
];

Typical symptoms

  • The repo never had a standalone .eslintrc file.
  • Teams need a deterministic config export before plugin cleanup.
  • Reviewers want to know whether multiple config sources block the easy path.

What the product covers

  • Static package.json eslintConfig discovery and reporting.
  • Generated FlatCompat bridge in the supported subset.
  • Manual-review findings for multiple config sources.

Manual-review boundary

  • Multiple ESLint config sources in one repo.
  • Existing flat config files.
  • JS-driven config logic outside static data.

Keep the migration narrow

Use the exact-problem guides as a triage layer, then decide whether the repo is inside the supported subset.
