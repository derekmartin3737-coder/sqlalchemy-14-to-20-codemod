Manual review

Keep the real migration decisions visible.

Flagged for engineering review

  • engine.execute(...) and transaction boundary choices
  • dotted loader paths and multi-hop string joins
  • removed Query helpers outside the documented subset
  • parse failures or ambiguous call shapes

Limitations

What this is for

Zipper Tools helps teams remove repeated legacy SQLAlchemy cleanup from the branch so reviewers can focus on semantic upgrade work. It is strongest when the scan finds many instances of the documented patterns.

What this is not

It is not a full SQLAlchemy 2.0 migration robot. It does not fake support for async migration, transaction redesign, or every Query to select() rewrite. Validation failures stay visible in the report.
