flatconfig-lift proof and fit

Public proof and fit boundaries for flatconfig-lift, including supported scope and exact-problem entry pages.

8 exact-problem pages

The proof page is tied to specific migration error pages, not a generic refactor promise.

3 fit signals

The product stays inside the supported subset and keeps fail-closed boundaries visible.

3 stop signals

The proof is also a way to say no before a buyer spends money on the wrong repo.

What this proof covers

  • Public proof covers static JSON and YAML configs from real public repos.
  • JS configs and existing flat config files are blocked instead of guessed through.
  • The bridge output stays close to the source config with machine-generated reporting.

What stays out of scope

  • Repos built around .eslintrc.js, .cjs, or .mjs logic.
  • Repos that already migrated to eslint.config.*.
  • Teams that want the tool to understand arbitrary plugin ecosystem behavior.
