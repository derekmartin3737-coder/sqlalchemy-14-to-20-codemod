Pydantic issue page

Fix simple @validator(...) patterns for Pydantic v2.

This page is for the narrow supported validator subset: direct imports, exact two-parameter classmethod signatures, and predictable decorator rewrites that can be validated instead of guessed.

  • Pack details
  • Also see BaseSettings
  • Also see pre root_validator

What changes

Before
@validator("email")
def normalize_email(cls, value):
    return value.strip().lower()

After
@field_validator("email")
@classmethod
def normalize_email(cls, value):
    return value.strip().lower()

Buying rule

Use the public fit scan first. Buy the pack only when the repo stays inside the documented validator subset and the manual bucket is still expensive enough to justify the one-time pack.
