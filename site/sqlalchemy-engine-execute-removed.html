SQLAlchemy issue page

If your upgrade is blocked by engine.execute(...), start with manual review, not wishful automation.

This is one of the highest-signal SQLAlchemy 2.0 breakages and one of the easiest places to oversell. sa20-pack does not claim to auto-rewrite engine.execute(...) because the safe follow-up depends on transaction boundaries, connection ownership, and runtime context.

What changed

Before
engine.execute(stmt)

Typical follow-up
with engine.begin() as conn:
    conn.execute(stmt)

That is only a common direction, not a universal rewrite rule.

Buying rule

Run the free scan first. If the repo is dominated by engine.execute(...), do not buy based on this site alone. Buy only when the report shows enough supported SQLAlchemy cleanup around it to make the pack worthwhile.
