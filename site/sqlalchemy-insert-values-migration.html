SQLAlchemy issue page

Fix insert(table, values=...) for SQLAlchemy 2.0 without hunting every DML call by hand.

This page targets the supported simple DML constructor keyword path: legacy insert, update, and delete calls that can be rewritten into chained statement builders when the shape is straightforward.

  • Pack details
  • SQLAlchemy migration tool overview
  • Also see select([..])

What changes

Before
insert(users, values={"name": name})
update(users, whereclause=users.c.id == user_id, values={"name": name})
delete(users, whereclause=users.c.id == user_id)

After
insert(users).values({"name": name})
update(users).where(users.c.id == user_id).values({"name": name})
delete(users).where(users.c.id == user_id)
