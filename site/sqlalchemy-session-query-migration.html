SQLAlchemy issue page

If you searched for session.query migration, narrow the exact breakage before you buy anything.

A lot of teams say "session.query is deprecated" when the real pain is one of a few narrower changes around the old Query API: Query.get(), string relationship joins, string loader paths, or broader query-shape rewrites. sa20-pack is intentionally not a full Query -> select() migration promise.

What people usually mean

  • session.query(Model).get(pk)
  • session.query(Model).join("relationship")
  • session.query(Model).options(joinedload("relationship"))
  • broader Query helper removal outside current pack scope

Buying rule

Run the free scan first. Buy the pack when the report shows supported patterns clustered together. Do not buy if your repo is mostly asking for a broad Query -> select() rewrite.
