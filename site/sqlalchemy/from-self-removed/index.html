What to do about Query.from_self() removal in SQLAlchemy 2.0

Treat Query.from_self() as a manual-review migration instead of pretending there is one safe automatic replacement.

Search intent

from_self removed sqlalchemy 2.0

from_self() is a real migration problem, but the replacement depends on query shape and result semantics.

Direct answer

What changes

Keep Query.from_self() in manual review. The right replacement depends on the surrounding query structure and what rows should come back.

Example

Before and after

Before

query = session.query(User).filter(User.active == True).from_self(User.id)
