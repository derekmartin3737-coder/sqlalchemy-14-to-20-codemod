Fix joinedload_all is not defined in SQLAlchemy 2.0

Replace joinedload_all usage with chained joinedload calls when the path is obvious.

Search intent

joinedload_all is not defined SQLAlchemy 2.0

joinedload_all was removed, but only the simple relationship paths are safe to rewrite mechanically.

Direct answer

What changes

Use chained joinedload(...) calls with mapped attributes for clear paths. Stop for aliases, contains_eager, and duplicate join interactions.

Example

Before and after

Before

query = session.query(User).options(joinedload_all("orders.items"))

After

query = session.query(User).options(
    joinedload(User.orders).joinedload(Order.items)
)

Typical symptoms

  • Importing joinedload_all fails after the upgrade.
  • Loader options still use dotted string paths.
  • The query also contains explicit joins or aliases.
