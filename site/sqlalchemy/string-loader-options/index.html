How to replace string loader options in SQLAlchemy 2.0

Replace joinedload("addresses") and similar string loader options with mapped attributes when the relationship root is obvious.

Search intent

joinedload string path sqlalchemy 2.0

Simple loader strings are cleanup work. Dotted paths and query-shape-dependent options are not.

Direct answer

What changes

For straightforward paths, replace the string with the mapped attribute, like joinedload(User.addresses).

Example

Before and after

Before

query = session.query(User).options(joinedload("addresses"))
query = query.options(selectinload("orders"))

After

query = session.query(User).options(joinedload(User.addresses))
query = query.options(selectinload(User.orders))

Typical symptoms

  • Loader options still depend on legacy string paths.
  • The easy cases are repetitive but not conceptually hard.
  • Teams need a clean stop signal before dotted or aliased paths.

What the product covers

  • Simple joinedload/lazyload/selectinload/subqueryload string cleanup.
  • Mapped-attribute replacements when the root entity is obvious.
  • Cross-links to the dotted-path and joinedload_all manual buckets.

Manual-review boundary

  • Dotted paths like joinedload("orders.items").
  • contains_eager and duplicate-join interactions.
  • Cases where the owning entity is not obvious from the local code.
